백엔드 시스템을 처음 만들어보는데...일단 클라이언트 요청과 응답 구조 자체가 처음에 이해가 잘 안되서 HTTP 통신 시스템 구조에서 부터 차근차근 공부해보았습니다.

주간 공부 시 교재 FastAPI - Database 미니퀘스트 실습으로 XAMPP, DBeaver를 사용해보면서 데이터 생성에 자신감이 생겨, 백엔드 구현 시 처음부터 구조를 main - router - controller - model 패턴으로 구분하여 시작했지만, 코드 상으로 데이터 연결에 어려움에 부딪혀 처참히 실패함.

그래서 다시 main.py에서 전체 코드 작성 후 구조 분리를 시행하였으나, 시간 부족으로 main - router 까지 밖에 분리하지 못함. 데이터 연결은 커녕 list로 fake data를 임시로 만들어 구현하였고, 데이터 연결을 안하니 그 나름대로 거기에 맞춰 코드를 작성을 하다보니 조금 혼란스러웠다. 추후에 MySQL 데이터를 연결하여 과제를 보완해야겠다는 생각.

LLM 호출과 관련해서는 수업 시에 진행했을 때는 얼추 알아듣는 줄 알았으나 전혀 내가 알아듣지 못했구나를 알게되었다. 알렉스가 올려준 client 예시 코드 1 ~ 4 를 하나하나 뜯어보면서 공부했고 내 코드로 적용을 해보았는데, 결국 적용을 잘 못해서 다른 분들의 과제 코드를 보면서 ollama의 chat 기능을 사용하여 어찌어찌 구현해봄. 

전체적으로 어떤 요청을 어떻게 받고, 이걸 받아서 뭘 return 하고, LLM 시스템을 구현할때는 내가 클라이언트 입장에서 뭘 보고 어떤 데이터를 요청해야하고 등 전체적인 흐름에 대한 이해도가 낮고, 확실히 백엔드라 그런지 직관적으로 이해하기가 어렵다보니 쉬운 코드 구현도 아직 많이 어렵다는 생각이 들었다. 큰 그림을 그리는 연습을 해보자 . !
